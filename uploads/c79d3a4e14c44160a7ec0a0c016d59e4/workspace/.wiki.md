# Project Summary
**LRE HOLDINGS** is a B2B web platform that streamlines inventory and logistics management, focusing on consignment inventory, financial settlements, and delivery logistics. It features an offline-capable driver interface for seamless operations without internet access, while ensuring security and user management through robust role-based access controls.

# Project Module Description
The project consists of several functional modules:
- **User Management**: Admins can create, update, and manage user accounts.
- **Inventory Management**: Track and manage inventory items, including their status and location.
- **Consignment Management**: Handle consignments, including creation, updates, and status tracking.
- **Delivery Management**: Manage deliveries, assigning drivers and tracking delivery statuses.
- **Financial Management**: Handle payments and financial statements related to consignments.
- **Audit Logging**: Maintain logs of changes made to critical data for accountability.
- **Authentication**: Supports email/password login using Supabase, with role-based routing to dashboards.

# File Description Inventory
- **backend/**: Contains server-side code, including API routes, database models, and business logic.
- **frontend/**: Contains client-side code, including React components, styles, and build configurations.
- **scripts/**: Contains scripts for seeding the database and other utility tasks.
- **src/lib/supabase.ts**: Contains configuration and utility functions for interacting with Supabase, including user role checks and error logging.

# Technology Stack
- **Frontend**: Next.js 14, React 18, TypeScript, Tailwind CSS
- **Backend**: FastAPI, PostgreSQL
- **Offline Support**: Dexie.js, PQueue
- **Additional Libraries**: @supabase/supabase-js, browser-image-compression, date-fns, recharts

# Usage
1. **Install dependencies**: Run `pnpm install` in both the `backend` and `frontend` directories.
2. **Build the project**: Execute `pnpm run build` in the `frontend` directory to create the production build.
3. **Run the backend server**: Use the command `uvicorn main:app --reload` in the backend directory.
4. **Run the frontend application**: Use `pnpm start` in the frontend directory.
